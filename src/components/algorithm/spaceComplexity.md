## 공간 복잡도
+	#### 문제에서 주어지는 입력 중 가장 메모리를 많이 사용하는 입력이 들어왔을 때 해당 문제의 메모리 제한을 넘기지 않고 코드가 동작할 수 있는지 파악해야 한다 이때 필요한 게 공간 복잡도이며 시간 복잡도와 같이 빅-오 표기법으로 나타낸다
+	#### 예를 들어 C++ 기준으로 `int = 4byte`이며 크기 2000짜리 int 배열을 만들면 `4byte * 2000 = 8000byte = 8KByte`이다,
+	#### 재귀 함수 호출이 많을수록 메모리 사용량도 증가하니 주의가 필요하다.
+	#### 시간과 공간(메모리)는 `trade-off`관계를 갖는 게 일반적이라 메모리를 적게 쓰는 프로그램은 느리고, 많이 쓰는 프로그램은 더 빠른 경향이 있다.


### trade-off
>트레이드오프(trade-off, tradeoff) 또는 상충 관계는 다른 측면(메모리의 사용량)에서 이득을 얻으면서 집합 또는 디자인의 품질, 양, 속성을 없애거나 잃어버리는 일이 수반되는 상황적 결정이다. 즉, 하나가 증가하면 다른 하나는 무조건 감소한다는 것을 뜻한다

### 재귀 함수
>하나의 함수에서 자신을 다시 호출하여 작업을 수행하는 방식으로 주어진 문제를 푸는 방법이다. 재귀 호출이나 되부름이라고도 한다.
